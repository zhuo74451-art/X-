▶️ 本地 Dry-Run 说明

以下命令全部在本地执行，不会触发任何外部请求：

一键 Pipeline

python scripts/run_pipeline_once.py

完整 Dry-Run（含发布模拟）

python scripts/run_autopublish_dryrun_once.py --limit 50

发布检查

python scripts/run_v001_portable_release_check.py

🚀 下一步部署路线

  1. 从 GitHub clone 仓库到目标机器
  2. git checkout x-automation-v0.1.0
  3. 运行 python scripts/run_pipeline_once.py 验证本地可用
  4. 运行 python scripts/run_v001_portable_release_check.py 确认完整性
  5. 如需接真实 X API：必须单独审批，配置 .env，设置 ENABLE_X_POST=true
  6. 如需定时运行：使用系统 cron / Task Scheduler，不创建项目内 daemon
  7. 部署静态报告：复制 demo_outputs/ 和 reports/ 到 Web 服务器
  8. 后续版本迭代从 main 分支拉取最新代码
