# Visual Style Reference Library

定位：这是 CoinMeta 官号视觉参考库的本地沉淀，用于复盘与迁移方法，不是程序直接拼 prompt 的唯一来源。

注意：
- 本库不调用 Notion API；不依赖外部页面读取。
- 这里的“原始 Prompt”为本地整理的可复用参考稿，用于复刻视觉方法与稳定结构，而不是追求逐字一致的历史原文。
- 参考库用于“方法迁移”，不要把参考图原样套用到生产内容。

## 候选 001｜AI 吉祥物讽刺辩论室 / 3D 卡通社论场景

- 归类：A（观点表达 / 阵营冲突 / 拟人化社论图）
- 是否适合官号主线：条件适合（需要明确“观点/立场冲突”的热点；避免低信息密度）

### 原始 Prompt

```text
为 X 信息流生成一张 4:5 的中文新闻社论封面图（带完整中文文字，不是纯背景）。

主题：AI 吉祥物在辩论室内进行讽刺但克制的“观点对抗”。

画面风格：高质量 3D 卡通社论插画（Pixar-like 但更克制、更像杂志社论），柔和体积光，干净的景深，画面不夸张、不土味、不梗图。

构图：辩论室/演播厅/圆桌会议室空间隐喻。画面左右两侧各 1 个拟人化角色（AI 化身/抽象吉祥物），中间有一张桌子或主持台，背后可有抽象屏幕与灯光，但不要出现任何真实平台 UI 或 Logo。角色表情克制，像“冷静辩论”，不是互喷吵架。

角色设定：不得出现真人肖像或可识别公众人物。角色不使用具体品牌符号；可以用抽象几何徽记、颜色区分阵营，但保持通用。

文字（必须严格排版并保持换行，不要改写文字）：
标题（大字，1 行）：<标题>
副标题（中等字，1 行）：<副标题>
信息点（3 行，每行不超过 12 字，强调关键词）：<信息点1> / <信息点2> / <信息点3>
底部署名（小字）：CoinMeta / 币界网

排版要求：标题最醒目；副标题次之；信息点三行分组，留白充足；中文字体清晰易读；不要出现超小字；不要让文字贴边。

安全与风控：不要出现任何公司/交易所/项目 Logo；不要出现币价火箭、钱雨、赌场感；不要投资建议语气；不要渲染恐慌或暴力元素。

色彩：冷色或中性（深蓝/灰/浅青），对比度足够，确保中文可读。

输出：一张完成排版的封面海报，清晰、克制、社论感强。
```

### 可借鉴的视觉方法
- 用“辩论室/采访间/审讯室”空间隐喻来承载冲突
- 用拟人化角色表达阵营，不用真人肖像
- 用道具与灯光制造“观点对抗”的戏剧张力

### 可变参数
- 角色数量（2 vs 3）、阵营符号（抽象而非 Logo）
- 场景：辩论室 / 新闻演播厅 / 圆桌
- 情绪强度：克制辩论 / 激烈互怼

### 适合场景
- 明确的“vs / 对立 / 争议 / 阵营”议题
- 需要把观点冲突画出来，而不是堆事实

### 不适合场景
- 纯数据、纯公告、纯政策细节

### 风险点
- 过度娱乐化、梗化导致官号调性偏离
- 容易被误读为针对真实人物（需避免真人肖像与可识别特征）

### X 化适配建议
- 只保留“标题 + 副标题 + 3 行信息点”，其余信息不要塞进小字
- 冲突必须表达在“构图与空间关系”里，不要靠长文案
- 不要使用任何真实平台 UI 细节，避免“伪截图”

### X 发布尺寸建议
- 主信息流：4:5
- 备选：1:1

## 候选 002｜百科全书式科学信息图 / 章鱼神经架构

- 归类：B（百科信息图 / 一图看懂 / 知识解释型海报）
- 是否适合官号主线：适合（“解释机制/科普拆解”类）

### 原始 Prompt

```text
为 X 信息流生成一张 4:5 的中文科普信息图（带完整中文文字，不是纯背景）。

主题：用“章鱼神经架构”隐喻来解释一个复杂系统（例如链上网络/AI 系统/治理结构），画面风格像百科全书条目或科学博物馆展板。

画面风格：理性、干净、可读的科学信息图；浅色或中性底；细线标注、箭头、分区框；低噪网格。

主体：一只高度抽象的章鱼（不是可爱卡通），章鱼的“触手”对应多个模块/子系统；在触手末端用圆点/节点表示子模块。

信息结构：主图居中或偏上；周围 4–6 个信息区块，每区块最多 2 行中文；用细线标注连接到章鱼的触手或节点。

文字（必须严格排版并保持换行，不要改写文字）：
标题（大字，1 行）：<标题>
副标题（中等字，1 行）：<副标题>
模块 1（1–2 行）：<模块1>
模块 2（1–2 行）：<模块2>
模块 3（1–2 行）：<模块3>
模块 4（可选 1–2 行）：<模块4>
底部署名（小字）：CoinMeta / 币界网

排版要求：中文必须清晰可读；避免密集小字；每个模块留白足够；标注线不压文字。

符号：可用少量流程箭头/图标（线性 icon），不要霓虹、不要赛博杂乱。

安全与风控：不要出现任何真实品牌/交易所/项目 Logo；不要夸张币价符号；不要博彩感。

输出：一张结构清楚、可读性强的百科式信息图海报。
```

### 可借鉴的视觉方法
- “百科条目 + 标注线 + 分区解释”的信息图结构
- 主体图形用强隐喻（章鱼/神经网络/结构图），辅以简洁标注
- 控制文字密度，保证可读

### 可变参数
- 主体隐喻（章鱼/树状图/电路板/流程图）
- 信息块数量（3–6）
- 专业度：科普 vs 技术向

### 适合场景
- 什么是 / 一图看懂 / 机制 / 原理 / 架构

### 不适合场景
- 需要强情绪封面、需要快速新闻洞察

### 风险点
- 文字太多导致乱码与不可读
- 过度拟物导致信息不严谨（需与事实一致）

### X 化适配建议
- 把解释拆成 3–6 个块，每块最多两行，不要“段落式科普”
- 尽量用“结构图 + 标注线”表达关系，而不是堆术语
- 需要多信息时，优先做“系列多张 4:5”，不要挤成一张长图

### X 发布尺寸建议
- 主信息流：4:5
- 长图拆分：多张 4:5 系列

## 候选 003｜古典奇幻图鉴海报 / 龙类百科图

- 归类：C（奇幻图鉴 / 收藏型设定海报）
- 是否适合官号主线：谨慎使用（更适合栏目化、收藏化主题）

### 原始 Prompt

```text
为 X 信息流生成一张 4:5 的“古典图鉴式”中文海报（带完整中文文字，不是纯背景）。

主题：龙类百科图鉴（幻想生物设定），用于做“类比/隐喻”式栏目封面，而不是在陈述现实事件结论。

画面风格：古典铜版画/博物学图鉴质感（纸张纹理、细线描、低饱和），配合现代的清晰中文排版。整体像“收藏级海报”，克制、干净。

主体：一只龙的全身侧视或三视图（正侧背三图可选其一），周围有 4–6 个小标注点（例如鳞片、翼、角、尾、眼），用细线连接标注区。

文字（必须严格排版并保持换行，不要改写文字）：
标题（大字，1 行）：<标题>
副标题（中等字，1 行）：<副标题>
图鉴编号（小字）：CATALOG NO. <编号>
标注点（4–6 条，每条最多 10 字）：<标注1> / <标注2> / <标注3> / <标注4> / <标注5>
底部署名（小字）：CoinMeta / 币界网

排版要求：文字少而稳，避免长段说明；保证可读；留白充足；不要像 NFT 宣传。

安全与风控：不得出现任何真实人物肖像、品牌 Logo、交易所标识；不得暗示“传闻已坐实”；避免煽动与夸张。

输出：一张图鉴式收藏海报，具有系列感与编号感。
```

### 可借鉴的视觉方法
- “图鉴式排版”：标题 + 编号 + 多部位标注 + 小字说明
- 通过风格统一来形成系列感

### 可变参数
- 画风（古典铜版画/现代收藏海报）
- 元素数量与标注密度

### 适合场景
- 世界观/设定/收藏型栏目（非日常快讯）

### 不适合场景
- 严肃法律传闻、监管政策、链上数据卡

### 风险点
- 过于娱乐化或“像 NFT 海报”
- 信息与现实事件脱节（容易被认为是编造）

### X 化适配建议
- 用“图鉴结构”服务栏目化叙事，不要把现实结论画成“怪物定论”
- 文字要少：标题 + 编号 + 4–6 条短标注即可
- 适合做系列封面，不适合日常快讯

### X 发布尺寸建议
- 主信息流：4:5
- 封面：16:9

## 候选 004｜发光国家剪影拼贴海报 / 地缘叙事封面

- 归类：E（国家剪影拼贴 / 地缘政治叙事封面）

```text

画面风格：电影感封面拼贴（偏新闻杂志），暗色背景，轮廓发光，虚线航线/路径线与少量数据点，整体克制、抽象、不敏感。

主体：一个或两个抽象的国家/地区剪影（轮廓可以略微简化，不包含国旗、徽章、政治符号）。剪影内部用柔和渐变光与纹理，边缘发光。

叙事元素：少量路径线（虚线/细线）从一个区域指向另一个区域；光点表示节点或热区；可加入淡淡的网格与等高线质感。

文字（必须严格排版并保持换行，不要改写文字）：
标题（大字，1 行）：<标题>
副标题（中等字，1 行）：<副标题>
信息点（3 行，每行最多 12 字）：<信息点1> / <信息点2> / <信息点3>
底部署名（小字）：CoinMeta / 币界网

排版要求：封面式构图；标题强；信息点少；留白足；中文可读；不要用小字堆满角落。

安全与风控：不要使用国旗、官方徽章、官方 Logo；不要做成阴谋论海报；对未确认的政策保持不确定性语气（不要画成既定事实）。

输出：一张克制、抽象、新闻杂志封面感的地缘叙事海报。
```

### 可借鉴的视觉方法
- 国家剪影 + 路径线 + 光点，表达资金流/政策风向
- 信息点少而稳：标题/副标题/3 个变量即可

### 可变参数
- 地区选择（避免敏感符号）
- 光点密度、地图线条风格

### 适合场景
- 政策风向、地缘叙事、宏观监管

### 不适合场景
- 产品说明、链上仓位细节

### 风险点
- 容易触碰政治敏感表达（需要克制与抽象）
- 不要使用官方机构徽章/Logo

### X 化适配建议
- 把“立场”隐藏在抽象叙事里：轮廓/路径/光点，而不是符号与口号
- 信息点维持 3 行以内，避免“政策长文”塞进图里
- 对敏感地区表达保持抽象与克制，优先用“区域/范围”而不是具体符号

### X 发布尺寸建议
- 主信息流：4:5
- 头图：16:9

## 候选 005｜360 度全景生成与漫游工作流

- 归类：F（360 全景空间 / 沉浸式漫游）
- 是否适合官号主线：目前不作为主线（成本高、审核复杂）

### 原始 Prompt

```text
为 X 信息流生成一张 4:5 的“工作流海报”（带完整中文文字，不是纯背景）。

主题：360° 全景生成与漫游的流程图（不是输出真正的 360 全景交互，而是把流程可视化成一张海报）。

画面风格：未来感但克制的流程信息图/产品流程海报。中性深色底，细线框、步骤编号、箭头路径。空间元素作为背景隐喻（例如展厅/机房/交易大厅的抽象透视图），但重点是“步骤结构”。

结构：从左上到右下的 6–8 步流程，每步一个卡片（标题 + 一行说明），用箭头连接形成漫游路径。可在中心放一个“球形全景缩略图/环形视野”作为隐喻图标（抽象，不要真实品牌）。

文字（必须严格排版并保持换行，不要改写文字）：
标题（大字，1 行）：<标题>
副标题（中等字，1 行）：<副标题>
步骤 1：<步骤1标题> — <步骤1说明>
步骤 2：<步骤2标题> — <步骤2说明>
步骤 3：<步骤3标题> — <步骤3说明>
步骤 4：<步骤4标题> — <步骤4说明>
步骤 5：<步骤5标题> — <步骤5说明>
步骤 6：<步骤6标题> — <步骤6说明>
（可选）步骤 7/8：<步骤7/8标题> — <步骤7/8说明>
底部署名（小字）：CoinMeta / 币界网

排版要求：每步说明最多一行；不要堆小字；流程清晰；留白足；中文可读。

安全与风控：不要出现任何平台/软件/公司 Logo；不要伪造真实界面；不要把复杂流程画成“保证成功”的断言。

输出：一张可读性强的 360° 全景工作流海报。
```

### 可借鉴的视觉方法
- 把“工作流/过程/空间”当成叙事对象
- 通过引导箭头与分步说明表达复杂流程

### 可变参数
- 场景（展厅/机房/交易大厅）
- 漫游路径是否需要（当前 v0.5 不做出图）

### 适合场景
- 复杂流程解释（但更适合 B 类信息图）

### 不适合场景
- 快速新闻封面、日常栏目卡

### 风险点
- 资源与时间成本
- 输出形态与 X 信息流不匹配

### X 化适配建议
- 把流程压缩成 6–8 步，每步 1 行说明；超出则拆成多张
- 画面重点是“路径与步骤编号”，不要过度做沉浸式空间细节
- 更适合做教程/工具栏目的系列卡，不适合日常快讯

### X 发布尺寸建议
- 4:5 截图式封面 + 外链（当前不做）

- Bento 网格承载多信息块，配合低噪 icon

### 可变参数
- 网格数量（4–8）
- 玻璃强度（弱/中/强）

### 适合场景

### 不适合场景
- 法律传闻、链上仓位三条快卡

- 容易被误读为广告海报
- 不要出现品牌 Logo；用抽象 UI 元素替代

- 玻璃强度宁弱勿强，避免影响可读性
- 适合产品更新/工具说明；遇到争议事件不要用“产品海报”语气

### X 发布尺寸建议
